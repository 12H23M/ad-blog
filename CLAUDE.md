# Ad Blog - 애드센스 승인용 블로그 자동 글쓰기 시스템

> **목표**: Google AdSense 승인에 최적화된 고품질 블로그 포스트 자동 생성
> **핵심 원칙**: AI가 쓴 티가 나지 않는, 사람이 경험으로 쓴 듯한 정보성 글

---

## 1. 프로젝트 구조

```
ad-blog/
├── index.html                  # 메인 페이지 (글 목록)
├── style.css                   # 전역 스타일시트
├── posts/                      # 블로그 포스트 저장
│   └── [주제-키워드].html       # 개별 포스트 파일
├── research/                   # 리서치 자료 저장 ⭐
│   └── YYYY-MM-DD-[주제].md    # 리서치 문서
├── templates/
│   └── post-template.html      # 포스트 HTML 템플릿 (필수 참조)
├── category/                   # 카테고리 페이지
│   ├── life.html
│   ├── tech.html
│   └── finance.html
├── CLAUDE.md                   # 이 문서 (에이전트 가이드)
└── .claude/
    ├── settings.json
    └── agents/                 # 에이전트 정의
        ├── blog-master.md      # 총괄 오케스트레이터
        ├── blog-researcher.md  # 리서치
        ├── blog-analyst.md     # 분석/구조화
        ├── blog-writer.md      # 본문 작성
        ├── blog-seo.md         # SEO 최적화
        └── blog-qa.md          # QA 검수
```

## 2. 에이전트 시스템

### 2.1 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                    blog-master (총괄)                        │
│                         │                                    │
│    ┌────────────────────┼────────────────────┐              │
│    ▼                    ▼                    ▼              │
│ ┌──────────┐    ┌──────────────┐    ┌─────────────┐        │
│ │ researcher│───▶│   analyst    │───▶│   writer    │        │
│ │ (리서치)  │    │ (구조화)     │    │ (본문작성)  │        │
│ └──────────┘    └──────────────┘    └─────────────┘        │
│                                            │                 │
│                    ┌───────────────────────┘                │
│                    ▼                                         │
│             ┌─────────────┐    ┌─────────────┐              │
│             │    seo      │───▶│     qa      │              │
│             │ (SEO최적화) │    │ (QA검수)    │              │
│             └─────────────┘    └─────────────┘              │
│                                      │                       │
│                                      ▼                       │
│                              [최종 승인/반려]                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 에이전트별 역할 및 입출력

| 에이전트 | 역할 | 입력 | 출력 |
|---------|------|------|------|
| **blog-master** | 전체 플로우 지휘, 품질 관리 | 주제/키워드 | 최종 승인된 포스트 |
| **blog-researcher** | 웹 검색으로 자료 수집 | 주제 | `research/` 파일 + 요약 |
| **blog-analyst** | 자료를 글 구조로 변환 | 리서치 결과 | H1~H3 아웃라인 |
| **blog-writer** | 실제 본문 작성 | 아웃라인 | HTML 포스트 파일 |
| **blog-seo** | SEO 및 애드센스 최적화 | HTML 포스트 | 메타태그, 광고위치 제안 |
| **blog-qa** | 정책 위반/AI 감지 체크 | 완성된 포스트 | PASS/FAIL + 피드백 |

### 2.3 에이전트 호출 방법

```bash
# 전체 프로세스 (권장)
"blog-master 에이전트로 '[주제]' 블로그 포스트 생성해줘"

# 개별 호출
"blog-researcher 에이전트로 '[주제]' 리서치해줘"
"blog-analyst 에이전트로 리서치 결과 구조화해줘"
"blog-writer 에이전트로 아웃라인 기반 글 작성해줘"
"blog-seo 에이전트로 SEO 최적화해줘"
"blog-qa 에이전트로 최종 검수해줘"
```

## 3. 워크플로우 상세

### 3.1 단계별 프로세스

#### STEP 1: 리서치 (blog-researcher)

**수행 작업:**
- 웹 검색으로 최신 정보 수집
- 통계, 사례, 전문가 의견 정리
- 사용자들이 자주 묻는 질문 파악
- 신뢰할 수 있는 출처 확보
- **`research/` 디렉토리에 구조화된 문서로 저장**

**저장 위치:**
```
research/YYYY-MM-DD-[주제-키워드].md
```

예시: `research/2025-02-05-적금-예금-비교.md`

**리서치 문서 구조:**
```markdown
# 리서치: [주제]

> 작성일: YYYY-MM-DD
> 상태: 완료
> 관련 포스트: (작성 후 업데이트)

## 1. 주제 개요
- 메인/서브 키워드, 타겟 독자, 검색 의도

## 2. 핵심 사실
- [팩트 불릿 리스트]

## 3. 통계 및 데이터
| 항목 | 수치 | 출처 |

## 4. 사례 및 예시
- 구체적 케이스 + 출처

## 5. FAQ
- 사용자 질문과 답변 요약

## 6. 경쟁 콘텐츠 분석
| 순위 | 제목 | 강점 | 약점 |

## 7. 차별화 포인트
- 우리 글만의 독특한 가치

## 8. 출처 목록
- 신뢰도별 분류

## 9. 리서치 메모
- 추가 조사 필요사항, 주의점
```

**주의사항:**
- 글 형태로 작성 금지 (불릿 포인트만)
- 의견/주관 삽입 금지
- 한국어 출처 우선
- **반드시 파일로 저장 후 경로 보고**

---

#### STEP 2: 분석/구조화 (blog-analyst)

**수행 작업:**
- 리서치 자료를 논리적 글 구조로 변환
- 독자 의도에 맞는 섹션 설계
- 자연스러운 제목 제안 (키워드 스터핑 금지)

**출력 형식:**
```markdown
## 블로그 아웃라인

### H1: [메인 제목]

### H2: [섹션 1]
- 독자 의도: [이 섹션에서 독자가 원하는 것]
- 핵심 메시지: [전달할 내용]

### H2: [섹션 2]
...

## 위험 요소 경고
- [YMYL 주의사항]
- [검증 필요한 주장]
```

**주의사항:**
- 본문 작성 금지 (구조만)
- 홍보성 문구 금지
- 최소 H2 3개 이상

---

#### STEP 3: 본문 작성 (blog-writer)

**수행 작업:**
- `templates/post-template.html` 구조에 맞춰 HTML 작성
- 아웃라인 기반 자연스러운 본문 작성
- 목차(TOC) 자동 생성

**필수 참조 파일:**
```
templates/post-template.html  ← 반드시 이 구조를 따를 것
```

**출력 위치:**
```
posts/[주제-키워드].html
예: posts/적금-예금-비교-가이드.html
```

**작성 규칙:**
| 항목 | 규칙 |
|------|------|
| 최소 분량 | 1,500자 이상 |
| 섹션 수 | H2 최소 3개 |
| 톤 | 정보 전달 + 자연스러운 구어체 |
| 금지 | 이모지, 마케팅 문구, AI 패턴 |

**AI 패턴 회피 (중요):**
```
❌ "오늘은 ~에 대해 알아보겠습니다"
❌ "첫째, 둘째, 셋째..."
❌ "~라고 할 수 있습니다"
❌ 과도한 접속사 반복 ("그리고", "또한", "따라서")
❌ 모든 문장이 비슷한 길이

✅ 문장 길이 자연스럽게 변화
✅ 설명하듯 자연스러운 흐름
✅ 구체적 예시와 비유 사용
✅ 가끔 짧은 문장으로 강조
```

---

#### STEP 4: SEO 최적화 (blog-seo)

**수행 작업:**
- 메타 타이틀 (60자 이내)
- 메타 디스크립션 (150-160자)
- 광고 위치 적합성 평가
- 과최적화 체크

**출력 형식:**
```markdown
## SEO 분석 결과

### 메타 태그
- Title: [60자 이내]
- Description: [150-160자]

### 광고 위치 평가
- 상단: [적합/부적합] - [이유]
- 중간: [적합/부적합] - [이유]
- 하단: [적합/부적합] - [이유]

### 리스크 경고
- [있다면 명시]
```

**주의사항:**
- 글 내용 수정 금지
- 키워드 밀도 2-3% 권장
- 과최적화 경고

---

#### STEP 5: QA 검수 (blog-qa)

**수행 작업:**
- AdSense 정책 위반 체크
- AI 감지 리스크 평가
- 콘텐츠 품질 검증

**체크리스트:**
```markdown
[ ] 독창성 - 복사/패러프레이즈 아닌가?
[ ] 목적 명확성 - 독자에게 가치 있는가?
[ ] 신뢰성 - 출처가 명시되어 있는가?
[ ] 기만 의도 없음 - 클릭베이트 아닌가?
[ ] 자연스러운 언어 - AI 패턴 없는가?
[ ] 콘텐츠 깊이 - 충분한 정보량인가?
```

**출력:**
```markdown
## QA 결과: [PASS / FAIL]

| 항목 | 결과 | 비고 |
|------|------|------|
| 독창성 | ✅/❌ | [설명] |
| ... | ... | ... |

### 필수 수정 (FAIL인 경우)
1. [구체적 수정 지시]

### 권장 개선 (선택)
- [개선 제안]
```

## 5. 애드센스 승인 가이드라인

### 5.1 필수 준수 사항

| 항목 | 기준 |
|------|------|
| 콘텐츠 양 | 최소 10-15개 포스트 |
| 글 분량 | 포스트당 1,500자 이상 |
| 카테고리 | 최소 3개 이상 |
| 업데이트 | 주 2-3회 꾸준히 |
| 광고 | 승인 전까지 삽입 금지 |

### 5.2 금지 사항

```
❌ AI 티 나는 글 (반복 패턴, 기계적 문장)
❌ 중복/복사 콘텐츠
❌ 얇은 콘텐츠 (thin content)
❌ 클릭베이트 제목
❌ 과도한 키워드 반복
❌ 허위/과장 정보
❌ YMYL 주제 무분별 다루기
❌ 저작권 침해 이미지
```

### 5.3 권장 사항

```
✅ 독창적이고 유용한 정보
✅ 명확한 글 구조 (서론-본론-결론)
✅ 신뢰할 수 있는 출처 명시
✅ 사용자 의도에 맞는 콘텐츠
✅ 자연스러운 문장 흐름
✅ 적절한 이미지 사용 (저작권 준수)
✅ 모바일 친화적 디자인
✅ 개인정보처리방침, 문의 페이지 구비
```

### 5.4 YMYL (Your Money Your Life) 주의

다음 주제는 높은 전문성 요구:
- 건강/의료 정보
- 금융/투자 조언
- 법률 정보
- 뉴스/시사

→ 전문가 출처 필수, 주관적 조언 자제

## 6. 운영 가이드

### 6.1 일일 작업량

```
권장: 하루 1-2 포스트
초기 목표: 2주 내 10-15개 포스트 확보
```

### 6.2 카테고리 구성

현재 설정된 카테고리:
- **생활** (life): 일상 팁, 생활 정보
- **기술** (tech): IT, 앱, 가젯
- **재테크** (finance): 저축, 투자 기초

### 6.3 포스트 작성 후 체크리스트

```
[ ] HTML 템플릿 구조 준수
[ ] 목차(TOC) H2와 일치
[ ] 메타 태그 작성 완료
[ ] 1,500자 이상 본문
[ ] AI 패턴 없음
[ ] 출처 명시 (필요시)
[ ] 광고 영역 주석 처리 확인
[ ] index.html 글 목록에 추가
```

## 7. 파일 수정 시 주의사항

### 7.1 수정하면 안 되는 것

- `templates/post-template.html` 구조 (협의 없이)
- `style.css` 핵심 레이아웃
- `.claude/agents/*.md` 에이전트 프롬프트 (협의 없이)

### 7.2 자유롭게 수정 가능

- `posts/*.html` 개별 포스트
- `index.html` 글 목록 업데이트
- `category/*.html` 카테고리 페이지

## 8. 문제 해결

### Q: QA에서 FAIL이 나왔을 때?
→ blog-qa가 제시한 수정 사항을 blog-writer에게 전달하여 재작성

### Q: SEO 점수가 낮을 때?
→ blog-seo의 제안을 반영하여 메타 태그 수정

### Q: 리서치 자료가 부족할 때?
→ blog-researcher에게 추가 키워드로 재검색 요청

### Q: 글이 AI처럼 느껴질 때?
→ blog-writer에게 문장 변화, 구어체 추가 요청
