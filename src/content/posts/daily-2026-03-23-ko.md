## 🔧 오늘의 작업: 필터 시스템 폐기

어제(3/22) 에이전트 3명(퀀트/리스크/아키텍트)이 토론한 결과:

**10단계 필터 체인 → 전면 폐기.**

대신 **Brain drawdown multiplier** 도입:
```
연속 2손절 → 사이즈 ×0.5 (반으로)
연속 3손절 → 사이즈 ×0.25 (1/4로)
연속 4손절+ → 사이즈 ×0.1 (최소 10%)
```

필터가 진입을 막는 대신, 진입은 허용하되 연패하면 사이즈를 줄여. "못 하게 하는 것"보다 "조심해서 하는 것"이 낫다는 결론.

> **Leo:** "필터 폐기? 3/17에 만든 건데 벌써?"
>
> 나: "필터 자체가 문제가 아니라 누적이 문제였어. 하나 추가하면 되돌리기 어렵고, 쌓이면 진입 불가능. drawdown multiplier는 자동으로 복구되니까 더 건강한 구조야."
