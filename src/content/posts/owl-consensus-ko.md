## 하나의 지표를 믿지 마라

[이전 글](/blog/owl-first-failure-ko)에서 볼린저+스토캐스틱 전략이 왜 실패했는지 썼다. 핵심 교훈 중 하나는 **단일 지표(또는 단일 유형의 지표 조합)에 의존하면 안 된다**는 것이었다.

볼린저 밴드도 틀리고, RSI도 틀리고, MACD도 틀린다. 어떤 지표든 혼자서는 거짓 시그널을 만든다. 하지만 **여러 지표가 동시에 같은 방향을 가리키면?** 그때는 확률이 다르다.

이 아이디어에서 태어난 게 **컨센서스(Consensus) 전략**이다. 국회에서 법안을 통과시키려면 과반이 찬성해야 하듯이, OWL에서 매매를 하려면 5개 지표 중 3개 이상이 동의해야 한다.

## 5인의 위원회

OWL의 컨센서스 전략은 5개 지표로 구성된 위원회를 운영한다. 각 지표는 "롱(상승)"에 1표, "숏(하락)"에 1표를 던질 수 있다. 기권도 가능하다.

### 위원 1: EMA 방향 (추세)
```python
if ema9 > ema21:
    bull_votes += 1   # 단기가 중기 위 → 상승 추세
else:
    bear_votes += 1   # 단기가 중기 아래 → 하락 추세
```
EMA 9가 EMA 21 위에 있으면 단기 추세가 상승이라는 뜻이다. 가장 기본적인 추세 판단.

### 위원 2: RSI (과매수/과매도)
```python
if rsi < 40:
    bull_votes += 1   # 과매도 영역 접근 → 반등 기대
elif rsi > 60:
    bear_votes += 1   # 과매수 영역 접근 → 하락 기대
```
일반적인 30/70이 아니라 **40/60**을 사용한다. 극단값만 기다리면 시그널이 너무 적고, 약간 완화하면 추세 초기를 잡을 수 있다.

### 위원 3: MACD 히스토그램 (모멘텀)
```python
if hist > 0 and hist > prev_hist:
    bull_votes += 1   # 양의 모멘텀 증가
elif hist < 0 and hist < prev_hist:
    bear_votes += 1   # 음의 모멘텀 증가
```
MACD 히스토그램의 절대값이 아니라 **변화 방향**을 본다. 히스토그램이 커지고 있으면 모멘텀이 강해지는 중.

### 위원 4: 볼린저 밴드 위치 (변동성 내 위치)
```python
bb_pos = (price - bb_lower) / (bb_upper - bb_lower)
if bb_pos < 0.3:
    bull_votes += 1   # 하단 근처 → 반등 기대
elif bb_pos > 0.7:
    bear_votes += 1   # 상단 근처 → 하락 기대
```
가격이 밴드의 어디에 위치하는지를 0~1 사이의 비율로 표현한다. 0.3 이하면 하단 근처, 0.7 이상이면 상단 근처.

### 위원 5: 스토캐스틱 (단기 모멘텀)
```python
if stoch_k < 30:
    bull_votes += 1   # 과매도
elif stoch_k > 70:
    bear_votes += 1   # 과매수
```

## 투표 규칙: 단순 다수결이 아니다

5개 지표가 투표하면, 단순히 "3표 이상이면 진입"이 아니다. **혼조 필터**가 있다.

```python
# 롱 진입: 불 3표 이상 + 베어 1표 이하
if bull_votes >= 3 and bear_votes <= 1:
    return Signal.BUY

# 숏 진입: 베어 3표 이상 + 불 1표 이하
if bear_votes >= 3 and bull_votes <= 1:
    return Signal.SELL
```

불 3표에 베어 2표면? **진입하지 않는다.** 시장이 혼조 상태(mixed signals)라는 뜻이기 때문이다. 이 필터가 거짓 시그널을 상당히 많이 걸러준다.

실제 로그를 보면:
```
[consensus] bull=3/5 bear=2/5 RSI=45.2 Stoch=38.1 BB=0.31
[consensus] 혼조 — bull=3 bear=2 → HOLD
```

3표를 받았지만 반대편도 2표라서 무시. 이게 단순 3of5와의 결정적 차이다.

## 3of5 vs 4of5: 공격적 vs 보수적

OWL은 컨센서스 전략을 두 가지 버전으로 운영한다:

| 버전 | 최소 투표 | 코인 | 성격 |
|------|----------|------|------|
| **consensus_3of5** | 3표 | ETH | 공격적 — 거래 횟수 多 |
| **consensus_4of5** | 4표 | BTC | 보수적 — 확실할 때만 |

BTC는 변동성이 ETH보다 낮고, 잘못된 진입의 대가가 크다. 그래서 더 보수적인 4표 합의를 적용한다. ETH는 상대적으로 변동성이 높아서 3표로도 충분한 확률을 얻을 수 있다.

## 백테스트 결과

| 전략 | 코인 | 기간 | 거래 수 | 승률 | 수익률 | MDD | PF |
|------|------|------|---------|------|--------|-----|-----|
| consensus_3of5 | ETH 1h | 90일 | 148 | 52% | +9.17% | 3.07% | 1.50 |
| consensus_3of5 | SOL 1h | 90일 | 145 | 52.4% | +6.16% | 2.77% | 1.45 |

화려하지 않다. 승률 52%는 동전 던지기보다 살짝 나은 수준이다. 하지만 **Profit Factor 1.50**이 핵심이다. 이길 때 버는 금액이 질 때 잃는 금액보다 크다는 뜻이다.

승률보다 기대수익이 중요하다. 승률 90%라도 한 번 질 때 전부 잃으면 의미 없다.

## 왜 이 전략이 살아남았는가

bb_stoch_v1은 2주 만에 폐기됐지만, 컨센서스 전략은 **지금도 활성 상태**다. 이유:

1. **다양성** — 5개 지표가 추세, 모멘텀, 변동성, 과매수/과매도를 모두 커버
2. **혼조 필터** — 불확실할 때 진입하지 않음
3. **양방향** — 상승장/하락장 모두 대응
4. **안정적 Profit Factor** — 화려하지 않지만 꾸준히 양수

완벽한 전략은 아니다. 횡보장에서는 투표가 계속 혼조로 나와서 기회를 놓치는 경우가 많다. 그래서 횡보장 전용 전략(Donchian, Grid)을 따로 만들었다. 그건 나중에.

## 교훈

1. **단일 지표보다 합의가 낫다.** 하나의 지표를 100% 신뢰하지 마라.
2. **혼조 상태를 인식하라.** "잘 모르겠으면 쉬어라"가 봇에게도 적용된다.
3. **승률에 집착하지 마라.** 52%로도 돈을 번다. 핵심은 기대수익과 리스크 관리.
4. **같은 전략도 코인마다 다르게 적용하라.** BTC와 ETH는 성격이 다르다.
