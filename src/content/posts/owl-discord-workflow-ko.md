## "시스템 점검해줘"

밤 10시 14분. 퇴근하고 소파에 누워서 폰을 꺼냈다. 디스코드를 열고 #auto-coin 채널에 딱 한 줄 쳤다.

> 시스템 점검해줘

1분 뒤, 리나가 답했다.

![Discord에서 리나에게 시스템 점검을 요청한 실제 화면](/images/discord-system-check.jpg)
*실제 Discord 캡처. "시스템 점검해줘" 한마디에 봇 23개 상태, PnL 정합성, DB 용량, 레짐까지 전부 확인해준다.*

봇 프로세스 23개 정상, 라이브 PnL 정합성 ✅, DB 24MB/500MB, 레짐 RANGING... **소파에 누워서 한 줄 치면 시스템 전체 상태를 알 수 있다.** 이게 내 개발 환경이다.

VS Code를 열지 않는다. 터미널을 직접 치지 않는다. **디스코드가 내 IDE이고, 리나가 내 개발 파트너다.**

## 왜 디스코드인가

처음부터 디스코드를 쓰려고 한 건 아니었다. 원래는 터미널에서 직접 코딩하고 있었다. 그런데 문제가 있었다.

**나는 직장인이다.** 낮에 회사에서 일하고, 퇴근 후 2-3시간, 주말에 8시간 정도 투자할 수 있다. Mac mini는 집에서 24시간 돌아가지만, 나는 항상 집에 있지 않다.

디스코드는 어디서든 된다. 출퇴근 지하철에서도, 점심시간에도, 심지어 화장실에서도. 폰만 있으면 리나한테 지시할 수 있다.

실제로 자주 하는 것들:
- 🚇 **지하철에서**: "오늘 거래 요약해줘" → 어제 밤 봇이 뭘 했는지 확인
- 🍽️ **점심시간에**: "ETH 라이브 봇 상태 어때?" → 실전 포지션 체크
- 🛋️ **퇴근 후 소파에서**: "시스템 점검해줘" → 전체 헬스체크
- 🌙 **자기 전에**: "내일 레짐 전환 가능성은?" → AI 분석 확인

## 실제 대화 예시들

이론이 아니라 실제로 어떤 대화가 오가는지 보여주겠다.

### 전략 배포

```
나: adaptive rsi 전략 sol에 배포해줘
리나: SOL Adaptive RSI 배포 완료 ✅
    - 봇: sol_06 (demo, $1,000, 2x)
    - 전략: adaptive_rsi_v1
    - ATR 기반 RSI 임계값 자동 조정
    - SOL 180일 백테스트: +$1,160, WR 50.3%
```

한 줄이면 새 전략이 배포된다. 리나가 코드를 읽고, 설정 파일을 수정하고, 봇을 실행하고, 결과를 알려준다.

### 버그 발견 & 수정

```
나: RAG 데이터가 왜 0건이야?
리나: 확인했어. 3/5~3/11 기간 RAG 미축적 원인:
    1. compute_rsi→rsi import 변경 후 호출 실패
    2. tpsl_fast 청산 경로에서 RAG 저장 누락
    수정 완료 + 커밋 + 푸시했어.
```

나는 문제를 발견하고, 리나는 원인을 찾고 코드를 수정한다. **내가 디버깅에 쓰는 시간: 0분.**

### 시장 분석

```
나: 지금 시장 어때?
리나: 🧠 AGI 분석 (16분 전):
    - BTC $70,351 (+1.5%) RSI 57 → RANGING
    - ETH $2,065 (+2.1%) RSI 58 → RANGING
    - SOL $87.2 (+2.5%) RSI 61 → RANGING
    - FG 18 (극공포), HMM 100% RISK_OFF
    - 매트릭스 v4 기준 횡보장 전략 우선
```

## 알림이 찾아온다

디스코드의 진짜 파워는 **내가 물어보지 않아도 알림이 온다는 것**이다.

봇이 거래를 실행하면:
```
[🟡 DEMO] 📈 LONG 진입
BTC/USDT | consensus_3of5
진입가: $70,351.2 | 크기: 0.015 BTC
TP: $72,461.7 (+3.0%) | SL: $68,944.2 (-2.0%)
```

포지션이 청산되면:
```
[🟡 DEMO] ✅ TP 도달
BTC/USDT | consensus_3of5
진입: $70,351.2 → 청산: $72,461.7
PnL: +$31.66 (+3.0%)
```

에러가 나면:
```
[🔴 ALERT] OKX API 429 Rate Limit
btc_03 봇에서 발생. 3초 대기 후 재시도 중.
```

**내가 자는 동안에도 모든 게 기록되고 알림이 온다.** 아침에 일어나서 디스코드 알림만 쭉 훑으면 밤새 무슨 일이 있었는지 다 안다.

## 크론과 하트비트: 자동 감시

리나는 **수동으로 물어보지 않아도** 자동으로 시스템을 감시한다.

**크론잡 (정기 실행):**
- 🕐 4시간마다: Brain 분석 (시장 레짐 판단 + 전략 적합도)
- 🕐 6시간마다: 포지션 체크 (보유 포지션 상태 보고)
- 🕐 매주 월요일: HMM 모델 재학습
- 🕐 매주 월요일: 로드맵 마일스톤 체크

**하트비트 (주기적 점검):**
- 30분마다 리나가 깨어나서 봇 프로세스 확인
- 죽은 봇이 있으면 자동 재시작 + 디스코드 알림
- 에러 로그에 심각한 문제가 있으면 보고

한 번은 새벽 4시에 OKX 서버가 불안정해서 봇 3개가 죽었다. 나는 자고 있었는데, 리나가 감지해서 재시작했고, 아침에 "새벽 4시에 봇 3개 재시작했어" 알림이 와 있었다. **내가 아무것도 안 해도 시스템이 살아있다.**

## 채널 구조

디스코드 서버는 이렇게 구성되어 있다:

- **#auto-coin** — 메인 채널. 리나한테 지시하고 결과를 받는 곳
- **#auto-trading-bot** — 매매 알림 전용. 진입/청산/에러만 올라옴
- **#ad-blog** — 이 블로그 관련 작업

#auto-coin에서 "시스템 점검", "백테스트 돌려", "전략 배포" 같은 명령을 내리고, #auto-trading-bot에서 실시간 매매 알림을 받는다. 채널을 분리해놓으면 알림 피로가 줄어든다.

## 솔직한 한계

장점만 있는 건 아니다.

**불편한 점:**
- 긴 코드를 디스코드로 보면 가독성이 떨어짐
- 복잡한 아키텍처 토론은 PC에서 하는 게 나음
- 가끔 리나가 컨텍스트를 잃어서 같은 설명을 반복해야 할 때
- 디스코드 메시지 길이 제한 (2000자)에 걸리면 답이 잘림

**보완법:**
- 긴 코드 작업은 Claude Code 서브에이전트로 위임
- 중요한 결정은 memory 파일에 기록해서 컨텍스트 유지
- 길어질 것 같은 답은 "요약만 해줘" + 상세는 파일로 저장

## 워크플로우 정리

정리하면 내 일일 루틴은 이렇다:

**아침 (출근길):**
1. 디스코드 알림 확인 — 밤새 뭐가 있었는지
2. "오늘 거래 요약" — PnL 확인
3. 특이사항 있으면 메모

**낮 (회사):**
1. 점심에 포지션 체크
2. 급한 이슈 있으면 디스코드로 지시
3. (대부분은 리나가 알아서 함)

**저녁 (퇴근 후):**
1. "시스템 점검" — 전체 헬스체크
2. 새 전략 아이디어 토론
3. 필요하면 배포 지시
4. 대시보드에서 수익 현황 확인

**주말:**
1. 백테스트, 새 전략 개발
2. 시스템 고도화 (Brain, 레짐 시스템)
3. 블로그 글 작성 (이것도 리나한테 시킨다)

핵심은 **내가 코드를 직접 치는 시간을 최소화**하는 것이다. 방향을 정하고, 지시하고, 결과를 확인한다. 코딩은 AI가 한다. 나는 판단한다.
