## RSI 30/70의 함정

RSI(Relative Strength Index). 트레이딩 입문자가 가장 먼저 배우는 지표다.

- RSI < 30 → 과매도 → 사라
- RSI > 70 → 과매수 → 팔아라

간단하다. 너무 간단하다. **그래서 안 먹힌다.**

OWL 봇에 RSI 30/70 고정값으로 Mean Reversion 전략을 넣고 돌렸다. 결과?

| 기간 | 거래 수 | 승률 | 수익 |
|------|---------|------|------|
| BTC 90일 | 42 | 38% | -$320 |
| ETH 90일 | 51 | 41% | -$180 |

**처참했다.** 왜?

횡보장에서는 RSI 30/70이 꽤 잘 맞는다. 근데 **강한 추세장**에서는 RSI가 70 위에서 한참을 머문다. "과매수니까 팔아라"를 따르면 불장 초입에서 숏을 치게 된다. 반대로 하락장에서는 RSI가 30 아래에서 바닥을 기어다닌다. "과매도니까 사라"를 따르면 계속 떨어지는 칼을 잡는다.

**고정값이 문제**였다. 시장은 매일 다른데 기준은 항상 같으니까.

## 아이디어: 변동성에 맞춰 RSI를 조절하면?

생각해보면 단순하다.

- 시장이 미친 듯이 흔들릴 때(고변동) → RSI가 극단값에 가야 의미가 있다 → **임계값을 넓혀라** (20/80)
- 시장이 조용할 때(저변동) → RSI가 살짝만 벗어나도 시그널이다 → **임계값을 좁혀라** (35/65)

변동성을 어떻게 측정하지? **ATR(Average True Range).** 14봉 기준 평균 변동폭이다. 이 값이 크면 시장이 출렁이는 거고, 작으면 조용한 거다.

## 구현: 3단계 변동성 레벨

ATR을 퍼센트로 환산해서 3단계로 나눴다:

```
atr_pct = ATR / 현재가 × 100

if atr_pct > 3.0:   # 고변동 — 시장이 미침
    rsi_low, rsi_high = 20, 80
elif atr_pct > 1.5:  # 중변동 — 보통
    rsi_low, rsi_high = 28, 72
else:                # 저변동 — 잠잠
    rsi_low, rsi_high = 35, 65
```

**이게 전부다.** RSI 계산은 똑같고, 비교하는 기준선만 시장 상태에 따라 바뀐다.

추가로 두 가지 필터를 넣었다:

### 볼린저 밴드 극단 확인

RSI가 임계값을 뚫었다고 바로 진입하면 또 당한다. **볼린저 밴드 바깥에 있는지** 한 번 더 확인한다.

```python
if rsi < rsi_low and price < bb_lower:
    # 진짜 과매도 — 롱 진입
elif rsi > rsi_high and price > bb_upper:
    # 진짜 과매수 — 숏 진입
```

RSI + BB 이중 필터. 두 지표가 동시에 "여기가 극단"이라고 말할 때만 진입한다.

### ATR 기반 동적 TP/SL

고정 퍼센트 TP/SL도 문제다. 변동성이 크면 SL이 금방 걸리고, 변동성이 작으면 TP에 영영 도달 못 한다.

```python
tp = entry_price ± (ATR × 2.0)  # 평균 변동폭의 2배
sl = entry_price ∓ (ATR × 1.2)  # 평균 변동폭의 1.2배
```

R:R(Risk-Reward) 비율이 자동으로 약 1.67:1이 된다. 시장이 출렁이면 TP/SL도 넓어지고, 조용하면 좁아진다.

## 180일 백테스트 결과

BTC/USDT 1시간봉, 180일:

| 지표 | 값 |
|------|------|
| 총 거래 | 278건 |
| 승률 | 50.3% |
| 총 수익 | **+$1,160** |
| Profit Factor | 1.30 |
| 최대 MDD | 8.2% |
| 평균 보유시간 | 14시간 |

**+$1,160. PF 1.30.** 기존 고정 RSI가 -$320이었던 걸 생각하면 극적인 반전이다.

승률이 50.3%로 반반인데 수익이 나는 이유? **TP가 SL보다 크기 때문이다.** 이길 때 $8.35 벌고 질 때 $5.01 잃으면, 승률이 50%여도 돈이 남는다.

### 변동성 구간별 성과

| 변동성 | 거래 수 | 승률 | PF |
|--------|---------|------|----|
| 고변동 (ATR>3%) | 89 | 53% | 1.42 |
| 중변동 | 134 | 49% | 1.25 |
| 저변동 (ATR<1.5%) | 55 | 48% | 1.18 |

고변동 구간에서 가장 잘 먹힌다. 시장이 과도하게 반응할 때 평균 회귀가 강하게 작동하기 때문이다.

## 실전 배포

백테스트 결과가 괜찮아서 **btc_05 봇에 배포**했다. 기존에 돌리던 `rsi_mr_v1`(고정 RSI)을 교체.

```
봇: btc_05
전략: adaptive_rsi_v1 (← rsi_mr_v1 교체)
모드: 데모
자본: $1,000
레버리지: 2x
타임프레임: 1h
```

배포 첫날, 바로 버그가 터졌다. `tp_pct` 속성이 없어서 15분마다 AttributeError. 전략 클래스에 기본값을 안 넣었던 거다.

```python
# 버그: tp_pct/sl_pct가 없음
class AdaptiveRSIStrategy:
    def __init__(self):
        # ATR로 동적 계산하니까 고정값 안 넣었음
        pass  # ← 여기가 문제

# 수정: 기본값 + ATR 갱신
class AdaptiveRSIStrategy:
    def __init__(self):
        self.tp_pct = 0.02   # 기본 2%
        self.sl_pct = 0.012  # 기본 1.2%
        # ATR 계산되면 덮어씌움
```

이게 봇 에러 루프의 30%를 차지하고 있었는데, 수정 후 클린.

## 고정 RSI vs Adaptive RSI

한눈에 비교:

| | 고정 RSI (30/70) | Adaptive RSI |
|---|---|---|
| 진입 기준 | RSI 30/70 고정 | ATR에 따라 20~35 / 65~80 |
| 추가 필터 | 없음 | 볼린저 밴드 극단 |
| TP/SL | 고정 % | ATR 기반 동적 |
| BTC 180일 | -$320, WR 38% | **+$1,160, WR 50.3%** |
| 강점 | 단순함 | 시장 적응 |
| 약점 | 추세장에서 학살 | 파라미터 3개 추가 |

## 교훈

1. **고정값은 게으른 선택이다.** 시장은 매일 다르다. 파라미터도 시장에 맞춰야 한다.
2. **ATR은 만능키에 가깝다.** TP/SL, 진입 기준, 레버리지 — 변동성 기반으로 조절하면 대부분 개선된다.
3. **이중 필터는 거래 수를 줄이지만 질을 높인다.** RSI만 쓰면 278건이 아니라 500건+ 거래가 뜬다. 절반을 걸러내는 게 수익에 도움이 됐다.
4. **백테스트 통과 ≠ 실전 성공.** 배포 첫날 기본값 누락으로 에러 루프. 코드가 맞아도 프로덕션은 다르다.
